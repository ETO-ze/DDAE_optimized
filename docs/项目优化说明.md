# 项目优化说明（相对原始 Notebook）

原始仓库只有一个 Notebook，能展示论文思路，但工程化使用存在明显痛点（环境/复现/批处理/数值稳定性）。
本重构版做了这些改动：

## 1) API 与兼容性
- 统一改为 `tensorflow.keras`（避免 `keras` / Keras 3 / TF 版本冲突）
- 优化器参数用 `learning_rate=`

## 2) 数值稳定性
原 Notebook 的相关系数分母可能接近 0，容易 NaN：
- 增加 `eps`
- 对相关系数 `clip([-1,1])`

## 3) 数据切窗与重建
- `patch_mode=trace`：逐道沿时间滑窗（推荐）
- 推理端采用 overlap-add（Hann 权重），减轻边界伪影

## 4) 可复现与可批处理
- YAML 配置 + CLI 脚本
- 自动输出 run 目录（含 config_effective.yaml + history.csv + best/final 模型）

## 5) 迁移学习更鲁棒
- 不再“按 layer index”复制权重，而是按 layer name + weight shape 匹配复制
  - 输入/输出维度不一致也能复制中间共享层（与原 Notebook 思路一致，但更稳）

## 6) 可选结构升级（下一步）
- 在配置中把 `model_type: dense` 改为 `conv1d`，可得到更强的“结构建模”能力
- 进一步升级到 2D patch + U-Net 也更符合地震时空相干特性（这部分建议另开分支做对比实验）
